typedef int(* VisPluginTransformPaletteFunc)(VisPluginData *plugin, VisPalette *pal, VisAudio *audio)

A transform plugin needs this signature to transform VisPalettes.

  • plugin Pointer to the VisPluginData instance structure.
  • pal Pointer to the VisPalette that is to be morphed. Only 256 entry VisPalettes have to be supported.
  • audio Optionally a pointer to the VisAudio, when requested.
Returns:
0 on succes -1 on error.

int visual_transform_set_video ( VisTransform *  transform,
VisVideo *  video 
)

Used to connect the target display it's VisVideo structure to the VisTransform.

Using the visual_video methods the screenbuffer, it's depth and dimension and optionally it's pitch can be set so the transform plugins know about their graphical environment and have a place to draw.

After this function it's most likely that visual_transform_video_negotiate needs to be called.

See also:
visual_video_new
visual_transform_video_negotiate
Parameters:
transformPointer to a VisTransform to which the VisVideo needs to be set.
videoPointer to a VisVideo which contains information about the target display and the pointer to it's screenbuffer.
Returns:
VISUAL_OK on success, -VISUAL_ERROR_TRANSFORM_NULL on failure.
