visual_size_t  datasize,
VisBufferDestroyerFunc  destroyer

Initializes a VisBuffer, it's safe to use visual_buffer_set_data_pair on a non heap allocated VisBuffer, however do not unref it when you're using this method.

The reason to do this can be performance because the VisObject initialization is skipped. Be very careful with this and only do it when you know what you're doing.

See also:
visual_buffer_new

bufferPointer to the VisBuffer that is to be initialized.
dataThe which the VisBuffer encapsulates.
datasizeThe size of the data (in bytes).
destroyerThe destroyer that is to be used on the data when the buffer is destroyed or the refcount reaches 0.
Returns:
VISUAL_OK on success, -VISUAL_ERROR_BUFFER_NULL on failure.
